# 🎬 Sistema de Gestão de Férias Inteligente
 
## 📝 Descrição do Projeto
Este projeto consiste em uma aplicação web de gestão de férias corporativas desenvolvida utilizando a plataforma Bubble.io com suporte de Inteligência Artificial como acelerador de desenvolvimento. O sistema foi projetado para automatizar processos de solicitação, aprovação e gerenciamento de férias de colaboradores, aplicando conceitos modernos de engenharia de software, segurança e governança de dados.
 
Desenvolvido como parte da disciplina de **Inteligência Artificial (2024.1)**, o sistema demonstra como ferramentas de IA podem auxiliar na criação de aplicações completas, enquanto o papel do desenvolvedor permanece essencial na revisão da lógica, segurança, modelagem de dados e otimização da aplicação.
 
A aplicação foi baseada no modelo gerado automaticamente pelo Bubble AI Generator, porém passou por um processo completo de refatoração e melhoria manual, garantindo responsividade, escalabilidade e aplicação das boas práticas de desenvolvimento profissional.
 
O projeto utiliza conceitos de **Dark Mode Minimalista** e design responsivo, oferecendo uma interface moderna, intuitiva e adaptada para diferentes dispositivos.
 
*Figura 1: Dashboard principal do sistema exibindo solicitações de férias, aprovações e gerenciamento de usuários.*
 
## 🚀 Tecnologias Utilizadas
* **Plataforma:** Bubble.io
* **Integração IA:** Bubble AI Generator
* **Banco de Dados:** Estrutura nativa Bubble Data Types
* **Autenticação:** Sistema de login Bubble Authentication
* **APIs:** Bubble Data API e API Connector
* **Ferramentas Complementares:** Miro, Notion e Google Docs
* **Design:** Flexbox Responsivo, Dark Mode Minimalista
* **Segurança:** Privacy Rules e boas práticas OWASP Top Ten para LCNC
 
## 📊 Resultados e Aprendizados
O projeto alcançou resultados sólidos em ambiente de testes, demonstrando eficiência na utilização de IA como suporte ao desenvolvimento de aplicações web modernas.
 
* **Automação Inteligente:** A IA do Bubble gerou rapidamente a estrutura inicial da aplicação.
* **Refatoração Profissional:** Ajustes manuais foram realizados para corrigir falhas de layout, lógica e permissões.
* **Segurança de Dados:** Implementação de regras de privacidade garantindo que usuários visualizem apenas seus próprios dados.
* **Escalabilidade:** Estrutura de banco otimizada utilizando relacionamentos entre tabelas sem dependência de listas excessivas.
* **Governança de Código:** Organização de workflows utilizando cores, notas e categorização lógica.
 
### Diferenciais Competitivos Implementados
* Sistema de aprovação automática baseado em regras.
* Dashboard inteligente para acompanhamento de férias.
* Controle de permissões avançado com Privacy Rules.
* Estrutura preparada para integração futura com IA via API Connector.
 
Além da implementação técnica, o projeto proporcionou aprendizado aprofundado em:
 
* Engenharia de software aplicada ao Low-Code.
* Segurança e privacidade em aplicações Bubble.
* Otimização de consumo de WUs (Workload Units).
* Estruturação de banco de dados escalável.
* Governança de workflows e documentação de lógica.
* Estratégias de mitigação de Vendor Lock-in.
 
## 🔧 Como Executar
1. Crie uma conta gratuita no Bubble.io.
2. Gere um novo projeto utilizando o Bubble AI Generator.
3. Configure os Data Types e Option Sets planejados previamente.
4. Revise os Workflows gerados automaticamente pela IA.
5. Configure as Privacy Rules na aba Data > Privacy.
6. Ajuste o design responsivo utilizando Flexbox.
7. Realize testes de segurança e validação de permissões.
8. Publique a aplicação utilizando o ambiente Bubble Hosting.
 
## 🔒 Segurança e Privacidade
O sistema foi desenvolvido seguindo o conceito de **Privacy by Design**, implementando regras restritas de visualização de dados.
 
* Exclusão de regras públicas padrão criadas automaticamente pela IA.
* Criação de permissões específicas baseadas no usuário autenticado.
* Proteção contra exposição indevida de dados.
* Aplicação de conceitos OWASP voltados para plataformas Low-Code/No-Code.
 
## 📈 Estratégia de Escalabilidade e Governança
Para evitar dependência excessiva da plataforma Bubble (Vendor Lock-in), foi planejada uma estratégia futura de exportação de dados utilizando a Data API do Bubble em formato JSON, permitindo eventual migração para tecnologias tradicionais como React.js e Node.js.
 
Além disso:
 
* Os workflows foram organizados por cores.
* Todas as lógicas complexas foram documentadas com notas internas.
* O sistema foi preparado para manutenção por outros desenvolvedores.
